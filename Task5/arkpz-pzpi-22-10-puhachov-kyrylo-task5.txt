Харків
1. РОЗГОРТАННЯ ПРОГРАМНОЇ СИСТЕМИ ТА ДЕМОНСТРАЦІЯ ЇЇ РОБОТИ
1.1 Мета роботи


Розгорнути програмну систему, що моніторить стан мостів за допомогою IoT-датчиків, та продемонструвати її роботу на хмарному хостингу SmarterASP.NET.


1.2 Хід роботи
1.2.1 Розгортання бази даних


Для зберігання даних, отриманих із датчиків моніторингу мостів, використано сервіс SmarterASP.NET для хостингу SQL Server. Основні кроки:
	Реєстрація на платформі SmarterASP.NET: Створено обліковий запис та активовано безкоштовний план.
	Створення бази даних: У панелі управління SmarterASP.NET створено нову базу даних SQL Server.
	Підключення до бази даних: Використано рядок підключення, наданий SmarterASP.NET, що містить дані про сервер, користувача та пароль.
	Перенесення даних: Для міграції даних використовувалися стандартні інструменти SQL Server Management Studio (SSMS). Було виконано експорт/імпорт даних.
	Тестування підключення: Проведено тестове підключення до бази даних із серверної частини додатка.


1.2.2 Розгортання серверної частини


Серверна частина, що відповідає за обробку даних із датчиків, була розгорнута на SmarterASP.NET:
•	Завантаження проєкту: Вебзастосунок було завантажено на платформу через FTP.
•	Конфігурація додатка: Налаштовано web.config для коректного підключення до бази даних та визначено змінні середовища.
•	Деплой додатка: Запущено ASP.NET Core додаток із використанням стандартних інструментів SmarterASP.NET.
•	Перевірка API: Виконано тестові запити до API, що обробляють дані датчиків та відправляють їх на клієнтську частину.


1.2.3 Інтеграція IoT-датчиків


Система використовує IoT-датчики для збору даних про стан мостів, зокрема:
•	Температура конструкцій
•	Вібрація та коливання
•	Деформації елементів мосту
Дані передаються на сервер у реальному часі, де вони зберігаються у базі даних та обробляються для подальшого аналізу.


1.3 Висновки


У результаті лабораторної роботи було успішно розгорнуто програмну систему моніторингу стану мостів на хмарному хостингу SmarterASP.NET. Налаштовано серверну частину, базу даних та забезпечено стабільну роботу системи з IoT-датчиками. Система готова до демонстрації та подальшого використання для моніторингу інфраструктурних об'єктів.
